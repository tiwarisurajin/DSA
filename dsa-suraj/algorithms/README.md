# Algorithms

Distinct from `patterns/` (which teaches *recognition* — "what technique does this problem need") and `implementations/` (which is organized by *data structure/topic*). This folder is for named, classic algorithms that stand on their own and get referenced across multiple topics — sorting algorithms, classic graph algorithms, string-matching algorithms.

Each algorithm doc (once populated) follows: theory → visualization → implementation → dry run → complexity → applications → pitfalls → interview tips → reusable template — the same depth bar as everything else in this repo.
