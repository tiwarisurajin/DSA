## The single most important skill this section teaches

Solving the problem is maybe 40% of a technical interview. The other 60% is: did you clarify the problem before coding, did you narrate your thinking, did you state complexity unprompted, did you test your own code, and did you handle being wrong gracefully when the interviewer pushes back. See [`technical/think_aloud_protocol.md`](./technical/think_aloud_protocol.md).
