# Notes

Freeform working notes — the scratchpad, distinct from the polished `docs/` and `implementations/` content. Things that live here: half-formed intuitions, questions to resolve later, links to explanations that clicked, things to come back and formalize into the Handbook once they're solid.

Nothing here is expected to be publication-quality. That's the point of the folder — the polished repo shouldn't need this, but the *work* of getting there benefits from a lower-stakes place to think.

Start a dated file per working session if you want structure, e.g. `2026-07-03_sliding_window_confusion.md`, or just keep a running `scratch.md` — this repo doesn't enforce a format here on purpose.
